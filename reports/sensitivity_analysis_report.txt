# AI Model Sensitivity Analysis Report

## 분석 배경
사용자로부터 "반응 온도를 변경해도 예측 결과가 변하지 않는다"는 피처 피드백을 수령하여 원인 분석을 수행함.

## 분석 결과

### 1. 온도 및 반응 시간 변수 (데이터 제약)
- 원인: 원본 데이터셋(Lab 합성 총괄) 확인 결과, 모든 실험의 반응 온도가 83.0도, 반응 시간이 4.75시간으로 고정되어 있음.
- 영향: AI 모델 학습 과정에서 변동이 없는 상수는 정보로서의 가치가 없으므로 피처 중요도가 0으로 처리됨. 이에 따라 입력값을 변경해도 예측 모델은 이를 반영할 수 없는 상태임.
- 조치: 현재 데이터셋으로는 해당 변수의 영향력을 예측할 수 없음을 사용자에게 안내. 향후 다양한 조건의 데이터가 추가되어 재학습될 경우 자동으로 활성화될 예정임.

### 2. 이론 고형분 변수 (수정 완료)
- 문제: UI에서는 48.0(wt%) 형식을 사용하나, 학습 데이터는 0.48(소수점) 형식으로 되어 있어 약 100배의 스케일 오차 발생 가능성 확인.
- 조치: app.py 내 추론 로직에서 입력값을 100으로 나누어 학습 모델의 스케일과 동기화함.

### 3. 피처 중요도 확인 (상위 5개)
- Tg: monomer_MMA(0.61), monomer_BMA(0.15), monomer_BA(0.11) 순
- 점도: monomer_AA(0.44), monomer_EMA(0.15), Scale(0.10) 순
- 수율: Scale(0.46), 이론 고형분(0.19), monomer_NDM(0.11) 순

## 결론
온도 변화 미반영은 시스템 오류가 아닌 학습 데이터의 본질적 한계(실험 조건의 고정)에 기인함. 고형분 스케일링 버그를 수정하여 전반적인 예측 신뢰도를 향상시킴.
